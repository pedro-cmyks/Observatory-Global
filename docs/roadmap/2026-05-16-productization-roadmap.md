# Atlas Productization Roadmap — 2026-05-16

## Current phase

Atlas has moved from issue-closing mode into productization mode.

The next phase is not "more panels." The next phase is making the existing intelligence legible, persistent, and exportable.

## Roadmap theme

**Make Atlas teachable, persistent, and exportable.**

- Teachable: users understand what each lens means and how to apply it.
- Persistent: users do not lose investigative context when they pivot.
- Exportable: users can turn discovered signals into dossiers, editions, and documented cases.

## Milestone 1 — Trust and continuity

Goal: remove the reasons a curious user loses confidence.

Issues:

- #136 Loading/prefetch across Landing -> Brief -> App.
- #137 Editorial analysis and methodology explanation.
- #138 Context preservation across pivots.
- #128 Workspace viewport and Trail/Pinned graph clarity.
- #104 Google Trends data freshness.
- #142 Public Attention scoping/actionability.
- #69 Spanish and multilingual search routing.

Exit criteria:

- Landing -> Brief -> App feels continuous.
- A Spanish query like `conflicto en Colombia` routes to a useful country/topic investigation.
- Theme-country pivots preserve the previous investigation.
- Trends/Public Attention either show relevant scoped data or an explicit useful empty state.
- Workspace can be opened and understood on a laptop viewport.

## Milestone 2 — Visual onboarding and use cases

Goal: help different user profiles reach value without accidental exploration.

Issues:

- #140 Visual use-case manual with annotated real Atlas screenshots.
- #134 Real-world analytical use cases.
- #135 Landing cards and live stats linked to docs/product flows.

Exit criteria:

- Landing use-case cards are clickable.
- Docs show real product screenshots, not generic text only.
- Each profile has at least one guided flow:
  - Journalist/editor: find story -> pin evidence -> dossier.
  - Investor/risk analyst: drift/sentiment -> country comparison -> watch.
  - Intelligence analyst: map signal -> source integrity -> temporal sequence.

## Milestone 3 — Evidence output

Goal: turn investigation work into something readable and reusable.

Issues:

- #133 Export consolidated signal dossier from Workspace nodes.
- #141 Reading Mode / Atlas investigation newspaper.
- #124 Saved watches / persistent alerts.

Exit criteria:

- Pinned Workspace items can generate an evidence dossier.
- Reading Mode can render a newspaper-style investigation from pinned evidence.
- Saved watches can become repeated monitoring inputs for future dossiers or Atlas Daily editions.

## Milestone 4 — Map and time polish

Goal: make advanced visual layers useful without overwhelming the user.

Issues:

- #139 Extend range beyond 24h and clarify Live/Pause.
- #143 Explain and declutter live map layers.
- #56 Improve day/night overlay and Settings utility.

Exit criteria:

- App supports at least a one-week analytical window where backend data allows it.
- Map layers are understandable and can be toggled with meaningful labels.
- Day/night overlay is visually softer and Settings explains useful controls.
