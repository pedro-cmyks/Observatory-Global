## New Feature: AI Coverage Insight

### backend/app/main_v2.py
New endpoint: `GET /api/v2/theme/{theme_code}/insight`

- Generates a 2–3 sentence meta-summary describing HOW a topic is covered (framing, tone, source patterns) — not a factual summary of the topic itself.
- Model: Claude Haiku (`claude-haiku-4-5`) via the `anthropic` SDK.
- Optional Ollama fallback: set `INSIGHT_PROVIDER=ollama` and `OLLAMA_HOST` to use a local model instead.
- Redis cached for 15 minutes. Cache key: `insight:{THEME_CODE}:{hours}`.
- Graceful fallback when `ANTHROPIC_API_KEY` is absent: returns `{"insight": null, "error": "insight_unavailable"}` without crashing.
- System prompt enforces pattern/statistical language only — no editorializing.

### backend/pyproject.toml
- Added `anthropic>=0.40.0` dependency.

### .env.example
- Added: `ANTHROPIC_API_KEY`, `INSIGHT_PROVIDER`, `OLLAMA_HOST`, `OLLAMA_MODEL`.

**To activate**: add `ANTHROPIC_API_KEY=sk-ant-...` to your `.env` file. Without it the endpoint returns `insight: null` silently — no other features are affected.
